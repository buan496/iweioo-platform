# Security policy

## Supported versions

The project is pre-release. Security fixes are applied to the latest version
on the `main` branch until the first stable release defines a longer support
policy.

## Reporting a vulnerability

Do not disclose a suspected vulnerability in a public issue, discussion, pull
request, or social-media post.

Use GitHub private vulnerability reporting for this repository. If that is not
available, contact `contact@iweioo.com` with a minimal description and a safe
way to continue the conversation. Do not include production secrets, personal
data, destructive payloads, or unnecessary exploit details in the first
message.

Reports should include the affected version, impact, reproduction conditions,
and any proposed mitigation. Receipt will be acknowledged when the reporting
channel is operational, followed by triage and coordinated disclosure.

Only test systems and data you own or are explicitly authorized to test.
